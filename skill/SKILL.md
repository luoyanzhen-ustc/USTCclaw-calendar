# Calendar Assistant Skill

**自然语言日历助手** - 课表识别、日程管理、智能提醒

## 触发条件

本技能在以下情况自动激活：

1. **用户上传了图片** → 自动检测是否为课表
2. **用户提到时间相关词汇** → 如"今天"、"明天"、"课"、"计划"等
3. **用户直接询问日程** → 如"今天有什么课"、"这周安排"

## 核心功能

### 1. 课表图片识别

当用户上传课表图片时：
- 自动调用 Qwen-VL 模型识别
- 提取课程名称、时间、地点、周次
- 向用户展示识别结果并确认
- 批量添加到日历

### 2. 日程查询

支持自然语言查询：
- "今天有什么课" → 显示今日日程
- "明天呢" → 显示明日日程
- "这周安排" → 显示周视图
- "下周三下午有空吗" → 查询特定时间段

### 3. 事件添加

支持自然语言添加：
- "明晚 7 点去图书馆" → 自动解析时间、标题
- "后天下午 3 点和朋友出去玩" → 添加临时计划
- "提醒我下周一交作业" → 添加带提醒的事件

### 4. 事件管理

- 修改事件： "把图书馆改成 8 点"
- 删除事件： "取消明天的计划"
- 设置提醒： "提前 1 小时提醒我"

### 5. 智能提醒

- 定时检查即将发生的事件（每 30 分钟）
- 通过 QQ/微信/当前聊天推送提醒
- 支持静默时段（23:00-08:00）
- 高优先级事件提前 1 天 +1 小时提醒

## 数据结构

### 事件格式

```json
{
  "id": "evt-001",
  "type": "course|plan|reminder",
  "title": "事件标题",
  "priority": "high|medium|low",
  "schedule": {
    "kind": "weekly|once|range",
    "weekday": 1-7,
    "startTime": "HH:MM",
    "endTime": "HH:MM",
    "date": "YYYY-MM-DD",
    "weeks": "1-15"
  },
  "location": "地点",
  "teacher": "教师/负责人",
  "notes": "备注",
  "reminderOffsets": [30, 1440],
  "active": true,
  "createdAt": "ISO timestamp"
}
```

### 存储位置

- `workspace/calendar/events.json` - 事件数据
- `workspace/calendar/settings.json` - 用户设置
- `workspace/calendar/reminders.json` - 提醒状态

## 意图识别规则

### 查询意图

| 模式 | 示例 | 意图 |
|------|------|------|
| `今天 (.*) 课` | "今天有什么课" | query_today |
| `明天 (.*) 安排` | "明天呢" | query_tomorrow |
| `周 (.*) 有` | "周三有课吗" | query_weekday |
| `这周 | 本周` | "这周安排" | query_week |

### 添加意图

| 模式 | 示例 | 意图 |
|------|------|------|
| `我要去 (.*)` | "我要去图书馆" | add_plan |
| `(.*) 点 (.*)` | "明晚 7 点学习" | add_plan_with_time |
| `提醒我 (.*)` | "提醒我交作业" | add_reminder |

### 确认意图

| 模式 | 示例 | 意图 |
|------|------|------|
| `好的 | 确认` | "好的" | confirm |
| `不要 | 不用` | "不用了" | cancel |

## 多渠道推送

### 当前聊天

始终启用，直接发送消息。

### QQ 推送

需要用户绑定 QQ 号，通过 QQBot 插件发送。

### 微信推送

需要用户绑定微信号，通过 WeChat 插件发送。

## 错误处理

- 图片识别失败 → 提示用户重新上传或手动输入
- 时间解析失败 → 询问用户澄清
- 推送失败 → 记录日志，尝试其他渠道
- 数据损坏 → 从备份恢复

## 扩展性

- 支持添加新渠道（邮件、短信等）
- 支持自定义视图模板
- 支持导出 ICS 文件
- 支持共享日历（小组作业）
