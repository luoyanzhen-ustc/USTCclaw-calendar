# 常见问题

## 安装相关

### Q: 安装后没有反应？

**A:** 检查以下几点：
1. 确认 Skill 已复制到正确目录：`~/.openclaw/workspace/skills/calendar-assistant`
2. 重启 OpenClaw：`openclaw gateway restart`
3. 查看日志是否有错误

### Q: 图片识别失败？

**A:** 可能原因：
1. 模型不支持图片识别 → 确认使用 `qwen-chat` 模型
2. 图片太模糊 → 重新上传清晰图片
3. 图片格式不支持 → 使用 PNG 或 JPG 格式

## 使用相关

### Q: 如何修改默认提醒时间？

**A:** 直接说：
```
设置默认提醒为 30 分钟
```

或者编辑配置文件：
```json
{
  "reminderDefaults": {
    "medium": [30]
  }
}
```

### Q: 如何关闭提醒？

**A:** 说：
```
关闭所有提醒
```

或者设置静默时段：
```
静默时段改为全天
```

### Q: 提醒没有推送怎么办？

**A:** 检查：
1. 是否在静默时段（默认 23:00-08:00）
2. QQ/微信插件是否已安装
3. 渠道配置是否启用

### Q: 如何删除已添加的事件？

**A:** 说：
```
取消明天的图书馆计划
删除后天的聚会
```

或者指定事件 ID：
```
删除事件 evt-001
```

## 课表识别

### Q: 课表识别不准确？

**A:** 尝试：
1. 重新上传清晰图片
2. 确保光线充足，避免反光
3. 手动修正识别结果
4. 或者手动输入课表

### Q: 课表有单双周怎么办？

**A:** 目前支持周次范围（如"1-15 周"），单双周标记正在开发中。可以手动修改：
```json
{
  "weeks": "单周"
}
```

### Q: 合并单元格识别错误？

**A:** 这是已知问题。建议：
1. 上传截图而非照片
2. 识别后人工核对
3. 手动修正错误课程

## 多渠道推送

### Q: 如何启用 QQ 推送？

**A:** 
1. 确保已安装 QQBot 插件
2. 说"启用 QQ 推送"
3. 绑定 QQ 群号或用户 ID

### Q: 如何启用微信推送？

**A:**
1. 确保已安装 WeChat 插件
2. 说"启用微信推送"
3. 绑定微信号

### Q: 可以推送到多个渠道吗？

**A:** 可以！说：
```
同时推送到 QQ 和微信
```

## 数据管理

### Q: 数据存在哪里？

**A:** 所有数据存储在本地：
- `workspace/calendar/events.json` - 事件数据
- `workspace/calendar/settings.json` - 配置
- 不上传云端，保护隐私

### Q: 如何备份数据？

**A:** 复制整个 `workspace/calendar/` 目录即可。

### Q: 可以导出为 ICS 吗？

**A:** 开发中！预计下个版本支持。

### Q: 可以同步到手机日历吗？

**A:** 开发中！预计后续版本支持 Google Calendar、Apple Calendar 同步。
